# 代码仓库搜索引擎 - 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品定位
一款基于AI工具的代码仓库搜索引擎，帮助开发者快速发现和探索优质的开源代码仓库。产品定位类似于Gitee的搜索功能，但专注于提供更智能、更直观的搜索体验。

### 1.2 目标用户
- 初级开发者：寻找学习参考的代码示例
- 中级开发者：查找特定功能的实现方案
- 高级开发者：发现优秀的开源项目和技术栈

### 1.3 核心价值
- **智能搜索**：利用AI技术理解用户意图，提供精准的搜索结果
- **可视化展示**：图文并茂的展示方式，快速了解项目信息
- **高效筛选**：多维度筛选和排序，快速找到目标仓库

---

## 2. 核心功能需求

### 2.1 关键词搜索功能 ⭐ 核心功能

#### 2.1.1 功能描述
用户可以通过输入关键词搜索代码仓库，系统返回相关的仓库列表。

#### 2.1.2 功能详情
- **搜索入口**：页面顶部提供搜索框
- **搜索方式**：
  - 支持仓库名称搜索
  - 支持描述内容搜索
  - 支持README内容搜索
  - 支持技术栈标签搜索
- **搜索建议**：
  - 输入时显示搜索建议（可选，初期可简化）
  - 支持热门搜索词推荐

#### 2.1.3 交互要求
- 支持回车键或点击搜索按钮触发搜索
- 搜索框支持清空功能
- 显示搜索历史（可选，初期可简化）

#### 2.1.4 优先级
**P0（必须实现）**

---

### 2.2 搜索结果分页展示 ⭐ 核心功能

#### 2.2.1 功能描述
搜索结果以分页形式展示，每页显示多个仓库信息。

#### 2.2.2 功能详情
- **分页参数**：
  - 每页显示数量：默认20个，可选10/20/50
  - 分页控件：显示当前页、总页数、上一页/下一页按钮
- **结果排序**：
  - 默认：相关度排序
  - 可选：按星标数、更新时间、创建时间排序
- **结果统计**：
  - 显示"找到 X 个相关仓库"
  - 显示搜索耗时

#### 2.2.3 交互要求
- 分页按钮支持点击跳转
- 支持直接输入页码跳转（可选，初期可简化）
- 加载状态提示（搜索中...）

#### 2.2.4 优先级
**P0（必须实现）**

---

### 2.3 图文样式搜索结果展示 ⭐ 核心功能

#### 2.3.1 功能描述
搜索结果以卡片形式展示，包含图片和文字信息，提供直观的视觉体验。

#### 2.3.2 功能详情
**每个仓库卡片包含：**

1. **仓库封面图**（可选）
   - 显示仓库的截图或Logo
   - 默认占位图（如果没有图片）
   - 尺寸：建议 300x200px

2. **仓库基本信息**
   - 仓库名称（可点击跳转）
   - 仓库描述（最多2行，超出省略）
   - 仓库所有者/组织名称
   - 仓库完整路径（如：username/repo-name）

3. **关键指标**
   - ⭐ 星标数
   - 🍴 Fork数
   - 👁️ 关注数（如果有）
   - 📅 最后更新时间

4. **技术标签**
   - 主要编程语言标签（如：JavaScript、Python）
   - 技术栈标签（如：React、Vue、Django）
   - 最多显示5个标签，超出显示"+N"

5. **操作按钮**
   - "查看详情"按钮
   - "收藏"按钮（可选，初期可简化）

#### 2.3.3 布局要求
- **卡片布局**：网格布局，响应式设计
  - 桌面端：每行3-4个卡片
  - 平板端：每行2个卡片
  - 移动端：每行1个卡片
- **卡片样式**：
  - 圆角设计，阴影效果
  - 悬停时轻微放大效果
  - 图片加载失败时显示占位图

#### 2.3.4 优先级
**P0（必须实现）**

## 6. 数据模型设计（简化版）

### 6.1 仓库表（repositories）
```
- id: 主键
- name: 仓库名称
- full_name: 完整路径（owner/repo）
- description: 描述
- owner: 所有者
- language: 主要语言
- stars: 星标数
- forks: Fork数
- created_at: 创建时间
- updated_at: 更新时间
- cover_image: 封面图URL
- readme_content: README内容（用于搜索）
- tags: 标签（JSON数组）
```

### 6.2 搜索记录表（search_logs）- 可选
```
- id: 主键
- keyword: 搜索关键词
- user_id: 用户ID（可选）
- created_at: 搜索时间
- result_count: 结果数量
```

## 7. API接口设计（核心接口）

### 7.1 搜索接口
```
GET /api/search
参数：
  - q: 搜索关键词（必填）
  - page: 页码（默认1）
  - per_page: 每页数量（默认20）
  - language: 语言筛选（可选）
  - sort: 排序方式（relevance/stars/updated，默认relevance）

返回：
{
  "total": 1000,
  "page": 1,
  "per_page": 20,
  "repositories": [
    {
      "id": 1,
      "name": "repo-name",
      "full_name": "owner/repo-name",
      "description": "仓库描述",
      "owner": "owner",
      "language": "JavaScript",
      "stars": 1234,
      "forks": 567,
      "updated_at": "2024-01-01",
      "cover_image": "https://...",
      "tags": ["react", "frontend"]
    }
  ]
}
```

### 7.2 仓库详情接口
```
GET /api/repositories/:id
返回：仓库详细信息
```

## 8. UI/UX设计要求

### 8.1 页面布局
- **顶部导航栏**：Logo、搜索框、导航菜单
- **搜索区域**：大号搜索框，居中显示
- **结果区域**：筛选栏 + 结果列表
- **底部**：分页控件

### 8.2 设计风格
- **现代简约**：扁平化设计，清爽的配色
- **加载状态**：骨架屏或加载动画
- **空状态**：无结果时的友好提示

### 8.3 交互细节
- 搜索框获得焦点时轻微放大
- 卡片悬停效果
- 平滑的页面滚动
- 友好的错误提示

---

## 9. 性能要求

### 9.1 响应时间
- 搜索响应时间：< 2秒
- 页面加载时间：< 1秒
- 图片加载：懒加载优化

### 9.2 并发支持
- 初期目标：支持100并发用户
- 可扩展至1000+并发

---

## 10. 风险评估与应对

### 10.1 技术风险
- **风险**：AI API调用成本高
- **应对**：初期使用缓存，限制调用频率

### 10.2 数据风险
- **风险**：数据来源不稳定
- **应对**：建立数据备份机制，考虑多数据源

### 10.3 法律风险
- **风险**：爬取数据可能涉及版权问题
- **应对**：遵守平台API使用规则，考虑官方合作
